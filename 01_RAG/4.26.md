# langgraph多线程通信：
## 为什么要用多线程通信？
- 并行化：需要同时运行多个 Agent 或任务，可能是通过多线程或多进程来执行。
- 共享数据：多个任务可能需要访问和修改共享数据或状态（例如：共享 state 或 evidence）。
- 异步操作：任务间需要等待异步响应（例如，等待 LLM 调用或外部 API 返回），这会涉及到同步机制

## 实现机制
### 1.异步编程asyncio
- 异步通信：LangGraph 内部很多操作是基于 asyncio 库的，尤其是 I/O 密集型任务，如调用 LLM、API 或数据库查询。这些操作本身是异步的。
- 事件循环：asyncio 提供了事件循环和任务调度机制，使得多个 Agent 的操作可以并行进行，但共享同一个线程。
```python
import asyncio

async def task1():
    await asyncio.sleep(1)
    print("Task 1 complete")

async def task2():
    await asyncio.sleep(2)
    print("Task 2 complete")

async def main():
    await asyncio.gather(task1(), task2())

# 启动事件循环
asyncio.run(main())
```
### 2.共享状态（State）与任务间通信
多任务间的数据共享是实现多线程通信的基础。在 LangGraph 中，状态通常会保存在 State 对象中，该对象可以被多个 Agent 共享，确保它们之间可以同步。
- state 传递：每个 Agent 都有一个 state 对象，该对象通过任务间的调用传递。例如，一个任务可能会从 state 中读取上一个任务的结果，并基于此做决策。
```python
async def agent1(state):
    state['result'] = "data from agent1"

async def agent2(state):
    data = state.get('result', 'default')
    print(f"Agent 2 received: {data}")

async def main():
    state = {}
    await asyncio.gather(agent1(state), agent2(state))

asyncio.run(main())
```
### 消息队列
在更复杂的多线程/多进程系统中，通常会使用 消息队列（例如 queue.Queue 或 asyncio.Queue）来进行任务间的通信。LangGraph 可能会在需要并行计算时使用这些消息队列来传递信息。
- 队列：一个线程或任务将数据放入队列中，另一个线程或任务从队列中取出数据进行处理。
```python
import asyncio

async def producer(queue):
    for i in range(5):
        await queue.put(i)
        print(f"Produced {i}")

async def consumer(queue):
    while True:
        item = await queue.get()
        print(f"Consumed {item}")
        if item == 4:
            break

async def main():
    queue = asyncio.Queue()
    await asyncio.gather(producer(queue), consumer(queue))

asyncio.run(main())
```
### 并发任务调度（Task Scheduling）
LangGraph 会使用任务调度器（如 asyncio.gather() 或其他调度工具）来管理多个并行任务。每个 Agent 的执行通常是异步的，而不同的任务（如 planner、supervisor、reflector）会被安排在不同的异步任务中执行。

- 任务调度：确保多个任务可以并行运行并根据顺序返回结果。
## 3.LangGraph 中的多线程通信实现原理
LangGraph 中的多线程通信依赖于上述机制（asyncio、消息队列、共享状态等），具体实现细节如下：

- 任务拆解与并行执行：
    - 例如，planner_node 生成多个子问题后，交给其他并行执行的节点（如 web_researcher、academic_researcher）去处理。
- 使用 asyncio.gather() 处理并行任务：
    - 在 asyncio 中，使用 gather() 函数同时调度多个异步任务，例如同时查询多个数据源。
- Agent 间共享 state：
    - state 对象在整个流程中传递，确保任务间的数据同步。例如，一个任务的结果（如 plan 或 evidence）将作为输入传递给下一个任务。
- 消息队列：
    - 如果任务之间需要更多的解耦，可能会使用消息队列来传递数据。例如，reflector_node 可能会等待来自不同 researcher 节点的结果，而这些结果会通过消息队列传递。
## 4.实际应用中的多线程通信
### 示例 1：并行执行与通信
在一个典型的 LangGraph 流程中，supervisor 可能会决定是否启动多个并行任务：
```python
async def supervisor_node(state):
    plan = state.get('plan')
    # 假设每个任务都是 async 函数，并且通过 asyncio.gather 并行执行
    tasks = [web_researcher(state, sq) for sq in plan]
    # *tasks 是 解包操作，把列表展开成单独的参数。
    # 相当于results = await asyncio.gather(task1, task2, task3)
    # 而不是gather(task) ==> gather([task1, task2, task3])
    # *解开列表/元组  **解开字典dict
    results = await asyncio.gather(*tasks)
    return results
```
这里，web_researcher 和其他任务会并行执行，并且通过 asyncio.gather() 等待它们的结果返回。所有的任务共享 state，因此它们能够在并行执行时同步数据。
### 示例 2：任务间的状态传递
```python
async def researcher_task(state):
    result = await fetch_data_from_api()
    state['research_result'] = result

async def supervisor_task(state):
    # 等待并获取研究任务的结果
    await researcher_task(state)
    print(state['research_result'])
```
在这里，researcher_task 会把数据写入共享的 state，而 supervisor_task 则从中读取数据并使用。
